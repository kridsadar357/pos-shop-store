# คู่มือการใช้งานแอปพลิเคชันเดสก์ท็อป POS Suite (ภาษาไทย)

แอปเดสก์ท็อปสร้างด้วย **Tauri** (ใช้เว็บวิวของระบบปฏิบัติการ ไม่ฝัง Chromium → ไฟล์เล็ก ~10 MB) โดยห่อหุ้มหน้าจอ POS เดิม (React) ไว้เป็นโปรแกรมติดตั้งได้บน **macOS / Windows / Linux**

แอปทำงานได้ 2 บทบาท เลือกตอนเปิดครั้งแรกใน “ตัวช่วยตั้งค่า”:

| บทบาท | ใช้เมื่อไหร่ | ทำอะไร |
|------|------------|--------|
| **เครื่องแม่ข่าย (Server)** | เครื่องหลักของร้าน | เปิดเซิร์ฟเวอร์ POS (API) ในเครื่องเอง + ต่อกับฐานข้อมูล PostgreSQL |
| **เครื่องลูกข่าย (Client)** | เครื่องขาย/แคชเชียร์เพิ่ม | เชื่อมต่อไปยังเครื่องแม่ข่ายผ่านเครือข่าย (LAN) |

```
┌───────────────────────┐      เครือข่าย LAN       ┌────────────────────────────┐
│  POS Suite (เครื่องลูก) │ ───────────────────────▶ │  POS Suite (เครื่องแม่ข่าย)   │
│  Tauri + หน้าจอ POS     │   /api (CORS_ORIGIN=*)   │  เซิร์ฟเวอร์ + PostgreSQL     │
└───────────────────────┘                          └────────────────────────────┘
```

---

## 1) สิ่งที่ต้องติดตั้งก่อน (Prerequisites)

ติดตั้งบนเครื่องที่จะ **พัฒนา/สร้างตัวติดตั้ง**:

1. **Node.js 20 ขึ้นไป** — https://nodejs.org
2. **Rust + Cargo** — https://rustup.rs (รันคำสั่งเดียวตามหน้าเว็บ)
3. **Tauri CLI**
   ```bash
   cargo install tauri-cli
   ```
4. ไลบรารีเว็บวิวของระบบ:
   - **macOS** — มีมาให้แล้ว (WKWebView) ไม่ต้องติดตั้งเพิ่ม
   - **Windows** — ต้องมี **WebView2 Runtime** (Windows 11 มีมาให้แล้ว)
   - **Linux** — ติดตั้ง `webkit2gtk` (เช่น `sudo apt install libwebkit2gtk-4.1-dev`)
5. **PostgreSQL** — จำเป็นเฉพาะ **เครื่องแม่ข่าย (Server)** (แอปไม่ได้ฝาก Postgres มาให้)

> ตรวจสอบว่าติดตั้งครบ: `node -v`, `cargo -v`, `cargo tauri -V`

---

## 2) รันแบบนักพัฒนา (Dev) — เปิดหน้าต่างแอปอย่างรวดเร็ว

ใช้ทดสอบหน้าจอ/ตัวช่วยตั้งค่า (เหมาะกับโหมด Client หรือเชื่อมต่อเซิร์ฟเวอร์ที่เปิดอยู่แล้ว)

```bash
# 1. รันเซิร์ฟเวอร์พัฒนาของหน้าเว็บ (Vite dev server ที่พอร์ต 5173)
npm --prefix web run dev

# 2. เปิดแอปเดสก์ท็อป (ในอีกหน้าต่าง Terminal)
cd desktop
npm run dev          # = cargo tauri dev
```

ครั้งแรกจะคอมไพล์ Rust สักครู่ จากนั้นหน้าต่างแอปจะเปิดขึ้นพร้อม **ตัวช่วยตั้งค่า** (ดูข้อ 4)

---

## 3) สร้างตัวติดตั้ง (Installer) — แบบครบในตัว (UI + เซิร์ฟเวอร์)

ใช้สำหรับติดตั้งจริงที่หน้าร้าน โดยเฉพาะ **เครื่องแม่ข่าย** เพราะจะฝังเซิร์ฟเวอร์ POS มาด้วย

```bash
cd desktop
npm run build        # = npm run bundle && cargo tauri build
```

- `npm run bundle` จะ build หน้าเว็บ + เซิร์ฟเวอร์ แล้วรวมเซิร์ฟเวอร์รุ่น production
  (โค้ด + Prisma + ไลบรารี + เครื่องมือฐานข้อมูล) ไว้ที่ `desktop/src-tauri/resources/server/`
- `cargo tauri build` จะรวมทั้งหมดเป็นตัวติดตั้ง

ไฟล์ผลลัพธ์อยู่ที่:
```
desktop/src-tauri/target/release/bundle/
   macOS   → dmg/POS Suite_1.0.0_*.dmg
   Windows → nsis/POS Suite_1.0.0_*-setup.exe
   Linux   → appimage/pos-suite_1.0.0_*.AppImage
```

> ⚠️ **ต้องสร้างตัวติดตั้งบนระบบปฏิบัติการเดียวกับที่จะนำไปใช้** เพราะเครื่องมือฐานข้อมูล
> (Prisma query engine) ขึ้นกับแพลตฟอร์ม — สร้างบน macOS สำหรับ Mac, บน Windows สำหรับ Windows

## 4) ตัวช่วยตั้งค่าครั้งแรก (Setup Wizard)

เมื่อเปิดแอปครั้งแรก จะให้เลือกบทบาทของเครื่อง:

### ก. เลือก “เครื่องลูกข่าย (Client)”
1. กรอก **ที่อยู่เซิร์ฟเวอร์** ของร้าน เช่น `http://192.168.1.50:4000`
2. กด **ทดสอบการเชื่อมต่อ** — ระบบจะลองเรียก `/health`
3. ถ้าสำเร็จจะบันทึกและโหลดเข้าหน้าเข้าสู่ระบบ

### ข. เลือก “เครื่องแม่ข่าย (Server)”
1. กรอก **DATABASE_URL** (การเชื่อมต่อ PostgreSQL) เช่น
   `postgresql://posadmin:pospassword@localhost:5432/posdb?schema=public`
   (เว้นว่างได้ ถ้าตั้งค่าไว้ในไฟล์ `.env` ของเซิร์ฟเวอร์แล้ว)
2. กด **เชื่อมต่อเครื่องนี้** — แอปจะชี้ไปที่เซิร์ฟเวอร์ในเครื่อง (`http://localhost:4000`)
   และครั้งต่อไปที่เปิดแอปจะ **เปิดเซิร์ฟเวอร์ POS ให้อัตโนมัติ** (ถ้าเป็นตัวติดตั้งที่ฝังเซิร์ฟเวอร์มา)

> เปลี่ยนการตั้งค่าภายหลังได้ที่หน้าเข้าสู่ระบบ → “**ตั้งค่าการเชื่อมต่อเซิร์ฟเวอร์**”

## 5) ตั้งค่าเครื่องแม่ข่ายให้รับเครื่องลูกข่าย (LAN)

เพื่อให้เครื่องลูกข่ายเชื่อมต่อข้ามเครื่องได้ เซิร์ฟเวอร์ต้องอนุญาต cross-origin
(ปลอดภัย เพราะระบบยืนยันตัวตนด้วย token ไม่ใช่ cookie):

```bash
# กรณีรันเซิร์ฟเวอร์เอง (ไม่ใช่ผ่านตัวติดตั้ง)
CORS_ORIGIN='*' npm --prefix server start
```
หรือกำหนดเป็นรายการที่อนุญาต เช่น `CORS_ORIGIN='http://192.168.1.51,http://192.168.1.52'`

วิธีหา IP ของเครื่องแม่ข่าย:
- macOS: `ipconfig getifaddr en0`
- Windows: `ipconfig` (ดู IPv4 Address)

แล้วเครื่องลูกข่ายกรอก `http://<IP เครื่องแม่ข่าย>:4000`

> วิธีที่ง่ายที่สุดในการเปิดเซิร์ฟเวอร์ + ฐานข้อมูล คือใช้ Docker:
> `docker compose -f docker-compose.prod.yml up -d` (ดู `DEPLOY.md`)

## 7) แก้ปัญหาที่พบบ่อย (Troubleshooting)

| อาการ | สาเหตุ / วิธีแก้ |
|------|----------------|
| เชื่อมต่อเซิร์ฟเวอร์ไม่สำเร็จ (Client) | ตรวจ IP/พอร์ตให้ถูก, เครื่องแม่ข่ายเปิดอยู่, อยู่วงเครือข่ายเดียวกัน, ไฟร์วอลล์เปิดพอร์ต 4000 |
| เครื่องลูกข่ายต่อไม่ได้ทั้งที่ ping ได้ | เซิร์ฟเวอร์ยังไม่ได้ตั้ง `CORS_ORIGIN=*` (ดูข้อ 5) |
| เครื่องแม่ข่ายเปิดไม่ขึ้น / ไม่มีข้อมูล | PostgreSQL ไม่ทำงาน หรือ `DATABASE_URL` ผิด — ตรวจการเชื่อมต่อฐานข้อมูล |
| ครั้งแรกเข้าหน้า `/setup` | เป็นการติดตั้งร้านครั้งแรก (ตั้งชื่อร้าน/ผู้ดูแล/ไลเซนส์) — ทำตามขั้นตอนบนจอ |
| `cargo tauri build` ล้มเหลวเรื่องไอคอน/รีซอร์ส | รัน `npm run bundle` ก่อน (สคริปต์ `npm run build` ทำให้อัตโนมัติแล้ว) |
| สร้างตัวติดตั้งแล้วใช้ข้ามเครื่อง OS ไม่ได้ | ต้องสร้างบน OS เดียวกับเครื่องปลายทาง (ดูข้อ 3) |

## 8) สรุปคำสั่งที่ใช้บ่อย

```bash
# ทดสอบ (dev) - ต้องเปิดสองหน้าต่าง/โปรเซสคู่กัน
# หน้าต่างที่ 1: รัน Vite dev server
npm --prefix web run dev
# หน้าต่างที่ 2: รัน Tauri desktop
cd desktop && npm run dev

# สร้างตัวติดตั้งครบในตัว
cd desktop && npm run build

# เปิดเฉพาะเซิร์ฟเวอร์ + ฐานข้อมูลด้วย Docker (ทางเลือกสำหรับเครื่องแม่ข่าย)
docker compose -f docker-compose.prod.yml up -d
```

> รายละเอียดเชิงเทคนิคของแอปเดสก์ท็อปดูเพิ่มได้ที่ `desktop/README.md`
> และการนำเซิร์ฟเวอร์ขึ้นใช้งานจริงดูที่ `DEPLOY.md`
